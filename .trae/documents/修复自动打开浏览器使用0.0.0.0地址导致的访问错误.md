## 问题分析

**报错信息**：浏览器显示"无法访问此页面"，错误代码 `ERR_ADDRESS_INVALID`，访问地址 `http://0.0.0.0:8000/`

**根本原因**：
1. 配置文件中 `MOCK_SERVER_HOST` 设置为 `0.0.0.0`，这是正确的服务器绑定地址
2. `0.0.0.0` 代表"所有网络接口"，但不是一个可直接访问的浏览器地址
3. 自动打开浏览器功能直接使用了服务器绑定地址 `0.0.0.0`，没有转换为可访问地址
4. 浏览器无法解析 `0.0.0.0` 地址，导致访问失败

## 解决方案

**修改 `simple_app.py` 中的 `open_browser` 函数**：

1. **添加地址转换逻辑**：
   - 当主机地址为 `0.0.0.0` 时，转换为 `localhost` 或 `127.0.0.1`
   - 保持服务器绑定地址不变，只修改浏览器访问地址

2. **修改自动打开浏览器逻辑**：
   - 在 `open_browser` 函数中添加地址判断
   - 将 `0.0.0.0` 转换为 `localhost`，其他地址保持不变

**具体修改**：
```python
def open_browser(host, port):
    """自动打开浏览器"""
    try:
        # 将0.0.0.0转换为localhost，便于浏览器访问
        browser_host = host if host != '0.0.0.0' else 'localhost'
        url = f'http://{browser_host}:{port}'
        # 等待服务器启动（给3秒延迟）
        time.sleep(3)
        # 使用默认浏览器打开
        webbrowser.open(url)
        logger.info(f"已自动打开浏览器，访问地址: {url}")
    except Exception as e:
        logger.error(f"自动打开浏览器失败: {e}")
        # 提示信息也使用转换后的地址
        browser_host = host if host != '0.0.0.0' else 'localhost'
        print(f"自动打开浏览器失败，请手动访问 http://{browser_host}:{port}")

# 同样修改启动前的打印信息
if not args.no_browser:
    # 转换打印的访问地址
    display_host = args.host if args.host != '0.0.0.0' else 'localhost'
    access_url = f'http://{display_host}:{args.port}'
    print(f"\n应用即将启动...")
    print(f"访问地址: {access_url}")
    print(f"按 Ctrl+C 停止应用\n")
```

## 预期效果

1. **服务器绑定**：仍然绑定到 `0.0.0.0`，支持多网卡访问
2. **浏览器访问**：自动打开浏览器时使用 `http://localhost:8000`
3. **用户体验**：用户无需手动修改地址，直接访问成功
4. **兼容性**：支持所有主流浏览器
